## 一、简介

在AI应用中,日志是核心可观测性数据源。通过结构化、语义化的日志指标,可快速定位问题、分析性能并评估系统健康度。

**产品demo直达**: https://evaplus.alibaba-inc.com/platform/analysis_insight/ai/trace?tenantId=1&projectId=95620731410383&projectType=AI_PROJECT

---

## 二、如何观测应用日志

在应用项目下,点击调用明细,可通过列表提供全链路可观测能力,支持模型开发与调优。

**例:**

- 通过单次请求trace的详细耗时,识别性能短板。
- 分析Token消耗、GPU利用率等,避免无效资源耗费计算。

可通过单次Trace请求的详情查看当前AI应用的调用工作流,如意图识别、MCP调用、数据预处理工作等,通过链路调用的观测,通过真实请求轨迹持续优化AI服务。

## 三、观测字段解释

| 指标                | 备注                                                                |
| ------------------- | ------------------------------------------------------------------- |
| Trace ID            | 一次会话的唯一标识符,用于追踪单次请求在分布式系统中的完整调用链     |
| 会话ID (Session ID) | 标识用户交互会话(如连续多轮对话),与单次请求(Trace ID)形成层级关系。 |
| Input/Output        | 记录原始输入内容(如用户问题)和模型输出结果(如生成文本)。            |
| Token数             | 输入提示词(input tokens) 生成结果(output tokens)的计算单元。        |
| 耗时                | 从请求发出到Ai应用响应返回的端到端时间(毫秒/秒级)                   |
| 节点Span            | 单个处理步骤(如模型推理、数据库查询)的耗时、状态、属性记录          |
