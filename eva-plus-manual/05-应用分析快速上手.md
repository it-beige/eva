## 一、简介

Eva+ 应用分析体系采用模块化分层架构,包含三大核心模块:AI 应用分析、AI 应用日志可观测、预警监控,形成"监测-诊断-预警-优化"的全链路闭环管理,确保 AI 应用可观测、可解释、可信赖。

其中,**AI 应用分析模块**细分为三个关键方向:

**产品demo直达**: https://evaplus.alibaba-inc.com/platform/analysis_insight/ai/apply-overview?spm=aplus2.7779473b.0.0.742c44adDJ4uuu&tenantId=1&projectId=95620731410383&projectType=AI_PROJECT

### 1.1 应用效果分析

量化评估应用在实际业务中的价值,通过应用调用量、响应时效、资源消耗等核心指标,实现AI应用效果的可量化分析与可监控管理。

### 1.2 用户行为分析

从用户行为视角出发,监控用户规模及行为路径,深度挖掘用户增长潜力与优化空间。

### 1.3 预警监控

基于关键性能指标设置预警阈值,实现问题的实时发现、精准定位与快速响应。

## 二、如何接入项目

### 2.1 在idealab&openlm发布的AI应用接入

如果您已在 Idealab 或 OpenLM 平台部署了 AI 应用,系统将为您自动完成项目接入,无需额外配置成本,即可无缝查看应用运行日志与分析报表,无需任何成本。

**平台示例说明:**

- **OpenLM 示例**: 当您在 OpenLM 发布一个语音识别模型服务后,系统会立即为其生成一个同名项目(如"Tisplus3"),并自动在Eva+平台生成项目对应的数据看板。

- **Idealab 示例**: 若您在 Idealab 创建了AI应用,Eva+中项目列表中会同步出现对应条目,日志观测数据和分析数据均可直接在控制台可视化呈现。

- **appbuilder示例**: 若您在 appbuilder中 创建了AI应用,Eva+中项目列表中会同步出现对应条目,点击数据入口-调用管理列下查看相关应用效果和模型数据。

- **自动创建项目**: 系统会为每个idealab&openlm(平台陆续新增中)创建的AI应用自动创建同名项目,并赋予应用Owner(创建者)项目管理员权限,支持编辑和删除操作。

- **查看项目列表**: 进入项目列表页,可查看所有有权限的项目。搜索功能:通过项目名称、应用Code或PID(项目唯一标识)快速定位项目。

- **添加数据看板权限**: 在项目管理页面,可将成员添加为普通用户,赋予其查看数据看板的权限。

### 2.2 集团内其他AI应用(非大模型平台发布)接入

- **新建项目**: 进入应用分析入口,点击"创建项目"。

- **创建项目的第一步**: 用于填写项目基础信息。

- **您可通过以下三种创建模式实现快速接入**:
  - **关联应用/场景**: 系统默认推荐"关联应用"模式,支持一键接入外部平台已有应用(如 Aone、TPP 等),接入后可直接查看应用数据。
  - **直接创建**: 若您尚未在其他平台接入应用,可直接新建应用。创建完成后,可通过 SDK 进行数据埋点。
  - **联合创建**: 适用于多应用数据合并分析的场景(如搜索场景中多个 AI 应用需统一监控)。您可选择【联合创建】项目,将多个应用整合至单一项目下进行数据观测。

- **填写项目名称、描述**,并指定管理员与普通成员,完成项目的初始化创建。

| 模块         | 说明                                                                                                            |
| ------------ | --------------------------------------------------------------------------------------------------------------- |
| 新建方式     | 提供三种创建模式:关联应用/场景(当前选中)、直接创建、联合创建。默认推荐"关联应用",支持一键接入外部平台已建应用。 |
| 平台选择     | 必填项,下拉菜单,选择要关联的第三方平台(如TPP AONE)。                                                            |
| 关联应用     | 必填项,下拉菜单,从所选平台中选择具体的应用或场景进行绑定,仅展示当前账号有权限的应用。                           |
| 项目名称     | 必填项,输入框,填写项目的唯一名称,便于识别和管理。                                                               |
| 项目描述     | 非必填,多行文本框,用于补充项目背景、目标或用途说明。                                                            |
| 管理员       | 必填项,可点击 × 移除,负责项目整体管理和权限配置。                                                               |
| 普通用户     | 非必填,搜索框,支持搜索并添加普通成员加入项目,具备基本操作权限。                                                 |
| 创建项目按钮 | 蓝色按钮,提交所有信息并创建项目,若必填项缺失则无法提交。                                                        |

- **埋点数据上报**:
  - 根据埋点规范,在应用内部集成PID维度的元数据上报功能。
  - 上报后,可通过系统观测AI应用的效果数据。

**埋点参考:**

- SDK接入: https://aliyuque.antfin.com/tqigeb/qm9roi/aundpgctdoolxy9g

## 三、如何看应用效果

### 3.1 应用分析

**(1)总览核心模块**

- **应用概览**: 展示应用调用、模型性能、用户满意度等核心指标。
- **用户行为**: 分析用户交互路径、活跃时段及功能使用偏好。
- **模型监测**: 跟踪模型推理速度、准确率及版本迭代效果。
- **API监测**: 监控系统响应时间、错误率及资源负载。

**(2)应用效果**

- 查看应用调用,应用token消耗,用户反馈评分等关键指标趋势。

**(3)用户行为**

- 分析用户规模、留存率、使用热力图。

**(4)模型监测**

- 评估模型消耗,调用等优化模型体验,监控延迟、错误日志请求情况。

**(5)API监测**

- 终端请求API的性能监测

## 四、关键术语说明

- **PID**: 项目唯一标识符,用于追踪项目下所有应用数据。
- **应用Code**: 应用的唯一代码标识。
- **埋点规范**: 需遵循的数据上报标准,确保元数据与平台兼容。

**SDK接入**:

- 终端用户行为SDK接入
- 服务端SDK接入

## 五、常见问题

- **Q:如何确保PID唯一性?**  
  A:系统默认生成唯一PID,手动修改时需检查是否重复。

- **Q:非平台应用如何接入?**  
  A:通过"新建项目"手动创建,完成PID配置并按规范埋点上报。

- **Q:数据权限如何分配?**  
  A:在项目管理页添加成员为"普通用户"或"管理员"。
