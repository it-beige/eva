Eva+中的评测功能帮助您从不同维度评测和衡量您的AI应用输出质量。

它提供了一个框架,用以系统性地通过数据集测试您的提示和模型,并使用多种指标来衡量性能。

Eva+还为常见的评测任务提供了一套**预构建的指标**。这些指标旨在帮助您快速有效地衡量您的 LLM 输出的性能,包括诸如幻觉 (Hallucination)、答案相关性 (Answer Relevance)、上下文精确率/召回率 (Context Precision/Recall) 等指标。

## 一、评测AI应用(agent)

### 1.1 分步指南

评测AI应用可以帮助掌握性能、功能表现,在评测任务可以查看怎么在平台上直接发起评测任务。

**评测过程分为以下几个核心步骤:**

1. 选择用于评测应用程序的评测集(Dataset)。
2. 定义评测任务(Evaluation task)。
3. 选择用于评测应用程序的指标(Metrics)。
4. 创建并运行评测任务(Evaluation experiment)。

**评测前的环境变量配置**  
点击平台右上角头像,可分别复制模型获取api-key和项目空间workspace-id。

### 1.2 运行离线评测

#### 1.2.1 选择评测集

```python
import os
from evaplus import Evaplus

os.environ["EVAPLUS_URL_OVERRIDE"] = "https://evaplus.alibaba-inc.com/api"
os.environ["EVAPLUS_WORKSPACE"] = "your-workspace-id"
os.environ["EVAPLUS_API_KEY"] = "your-api-key"

# Create a dataset
client = Evaplus()
dataset = client.get_or_create_dataset(name="My dataset")

# Add dataset items to it
dataset.insert([
    {"user_question": "Hello, world!", "expected_output": "Hello, world!"},
    {"user_question": "What is the capital of France?", "expected_output": "Paris"},
])
```

更详细内容可以参考: https://aliyuque.antfin.com/tqigeb/qm9roi/kpfpqqq6d94ha0p9

#### 1.2.2 定义评测任务

```python
def evaluation_task(dataset_item):
    # your LLM application is called here
    result = {
        "input": dataset_item["user_question"],
        "output": dataset_item["expected_output"],
    }
    return result
```

#### 1.2.3 选择评测指标

Eva+提供了一套内置的评测指标供您选择,主要分为两大类:

1. **code类指标 (Heuristic metrics)**: 本质上是确定性的,例如 equals(等于)或 contains(包含)。
2. **LLM裁判指标 (LLM-as-a-judge)**: 使用 LLM 来判断输出质量;常用于检测幻觉(hallucinations)或上下文相关性(context relevance)。

## 二、评测Prompt

### 2.1 分步指南

在开发Prompt和进行Prompt工程时,很难知道一个新Prompt是否比上一个版本更好。

Eva+允许在多个样本上评测Prompt,对每个 LLM 输出进行评分,并比较不同Prompt的性能。

**在 Opik 中,有两种评测Prompt的方法:**

1. 使用Prompt Playground
2. 在 Python SDK 中使用 evaluate_prompt 函数

### 2.1.1 使用Prompt Playground

Prompt Playground允许快速测试不同的Prompt并查看它们的表现。

您可以通过点击Playground右上角的添加PROMPT按钮来比较多个Prompt。

为了在样本上评测提示,您可以使用 {{variable}} 语法向Prompt消息中添加变量。然后选择一个评测集,并在每个评测集项上运行这些Prompt。

### 2.1.2 以编程方式评测提示

Eva+ SDKs 提供了一种使用 evaluate_prompt 方法评测提示的简单方式。

### 2.1.3 使用自定义模型自动化评分

手动审查每个 LLM 输出既耗时又容易出错。evaluate_prompt 函数允许您指定一个评分指标列表(scoring metrics),从而可以对每个 LLM 输出进行评分。
