# 基础概念

## 可观测性层级体系

**PID → 应用Code → Conversation → Trace → Span**

形成从业务项目到原子执行的可观测性层级体系。

此模型为日志聚合、请求链路可视化、性能瓶颈诊断及用户体验回溯提供了标准化。

### 层级说明

| 层级             | 说明                                                                                               |
| ---------------- | -------------------------------------------------------------------------------------------------- |
| **PID**          | 代表一个项目，可以是一个应用，也可以是多个应用的结合。                                             |
| **应用Code**     | 应用的唯一代码标识，支持统一身份ID。                                                               |
| **Conversation** | 对应AI应用与用户的完整交互流程，作为多条Trace的容器。通过Conversation ID与Trace关联。              |
| **Trace**        | 表示AI应用的一次执行过程，是多个Span的集合，也是平台的基本单元。通过Trace ID与其他元素如Span关联。 |
| **Span**         | 指代AI应用中的一个工作单元或模块，是最小执行单位。支持嵌套，并可通过Span ID关联其他信息。          |
