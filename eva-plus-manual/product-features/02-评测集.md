# 评测集

---

在开始执行评测实验前，需要先准备好评测集。本文指导你在 Eva+ 中创建和管理评测集。

## 一、评测集概述

评测集是用于系统化评估对象性能的标准化测试数据集。

评测集通常包含输入数据、预期输出和实际输出。输入数据作为评估对象的测试输入，预期输出提供评估基准，实际输出作为评估的内容。

### 按评测集类型分：

- 文本类评测集
- 代码类评测集
- 图片类评测集等

### 按评测集内容分：

- 至少包括：评测问题(input) 或 评测case
- 还可包括：评测实际结果(output)、正确答案(expected_output)、参考上下文(context)、会话多轮信息、节点调用信息、评测问题分类标签等

### 评测集创建方式对比：

| 创建方式       | 适用场景                                                                                | 状态   |
| -------------- | --------------------------------------------------------------------------------------- | ------ |
| 本地上传       | 适用于评测数据存在本地时                                                                | 已支持 |
| ODPS上传       | 适用于评测数据来自odps数据表时                                                          | 已支持 |
| 线上数据提取   | 适用于评测数据来自线上真实对话时(仅支持在idealab、openlm、appbuilder平台上搭建的AI应用) | 已支持 |
| SDK动态接入    | 适用于评测数据来自线上真实对话，但应用未在idealab/openlm/appbuilder上搭建               | 已支持 |
| AI智能生成     | 适用于给出几个示例后让AI参考生成                                                        | 已支持 |
| 空白评测集     | 适用于评测集还在筹备中，想先建个空的、后续再往里添加                                    | 已支持 |
| 引用公共评测集 | 适用于想使用业内公共的代码评测数据时                                                    | 已支持 |
| 自定义评测集   | 适用于评测数据来自gitlab上的project时                                                   | 已支持 |

## 二、创建评测集

**操作入口**：Eva+ > 应用评测 > 评测集 > 新建评测集

### 2.1 「文本类」本地上传

- Step1. 填写你希望创建的评测集名称及描述
- Step2. 上传你的评测集数据（注意：上传文件格式需要是CSV UTF-8，不然可能出现中文乱码。提示：你可以将数据写入excel表格、在保存时选择存为CSV UTF-8的格式。）

### 2.2 「文本类」ODPS上传

- Step1. 填写你希望创建的评测集名称及描述
- Step2. 填写需要上传的odps表名（注意：填写后Eva+会对该表进行权限校验；若Eva+的odps账号没有当前表中任何字段的权限，则需要用户按照帮助文档先对Eva+的odps账号赋予当前表中部分或全部字段的读取权限，再重新在此填写表名，通过权限校验环节。）
- Step3. 可选填写需要上传的odps表分区（若要一次限定多个分区，可使用英文逗号分隔）

### 2.3 「文本类」线上数据提取

- Step1. 填写你希望创建的评测集名称及描述
- Step2. 选择你希望从线上提取的数据粒度（注意：目前仅支持traceID粒度、即一问一答的单论对话粒度。如果有需要支持会话粒度、即一个会话窗口内的全部轮次，或有需要支持节点粒度、即一轮对话中的每个调用节点，可以联系我们）
- Step3. 选择你希望从线上提取哪段时间产生的数据（注意：目前已支持今日实时和历史14天的线上数据）
- Step4. 如有需要，还可添加筛选过滤条件；目前可以支持按照线上trace粒度数据的这些维度尽享筛选过滤「TraceID，用户ID，归属的会话ID，输入内容，输出内容，trace完成状态」（注意：一个评测集最多支持2万条数据）

> ‼️ Eva+的线上数据提取功能，默认提取的是Trace粒度的数据（即每条Trace的Input, Output, Token, Time等），如果需要更多自定义数据（例如Trace里全部的SpanType, 特定Span的Output等）我们也支持数据自定义提取。

**1）调用http服务接口**

- 海外新加坡API地址：https://sg-evaplus.alibaba-inc.com/api/v1/private/datasets/

### 2.5 「文本类」AI智能生成

- Step1. 填写你希望创建的评测集名称及描述
- Step2. 给出几行评测集示例数据：💡由于我们需要知道你希望有哪些数据列、每一列及每一行示例数据的内容关系，所以这里需要你将示例数据写入excel表格、存为CSV UTF-8的格式上传给我们，方便模型理解并构建相同结构的评测集。
- Step3. 选择模型，设置生成条数

### 2.6 「文本类」空白评测集

- Step1. 填写你希望创建的评测集名称及描述
- Step2. 设置你希望评测集中有哪些列（也可以跳过，等后续需要的时候再增删列）

### 2.7 「代码类」引入公共评测集

- Step1. 选择业内通用的代码领域公共评测集（目前已支持"terminal-bench"；更多正在引入中，敬请期待）
- 了解terminal-bench：https://code.alibaba-inc.com/evaplus-datasets/terminal-bench

### 2.8 「代码类」自定义评测集

- Step1. 填写你希望创建的评测集名称及描述
- Step2. 目前仅支持从gitlab引入代码类评测集，因此需要用户填写需要引入的gitlab中的project名称

## 三、编辑评测集

### 3.1 表管理

#### 3.1.1 列操作

- 点击图标可以新增列、修改列名
- 点击图标可以调整列排序、隐藏部分列
- 点击图标可以使用AI能力基于现有列生成新的列，多用于对现有列进行打标

#### 3.1.2 行操作

- 点击图标可以新增行（即继续添加数据）
- 点击图标可以调整行高（即每页展示行数）
- 点击图标可以使用AI能力基于现有行生成新的行，多用于对现有数据进行扩充

#### 3.1.3 全局操作

- 点击图标可以筛选数据
- 点击图标可以对筛选出的、或勾选出的数据进行另存或删除
- 点击图标可以对当前评测集增加分类标签

### 3.2 行新增

#### 3.2.1 手动添加行

在评测集详情页面，选择新增数据项 > 手动添加来添加评测数据

- 点击后右侧推出抽屉，由用户填写
- 如果当前评测集中已有数据，平台会根据已有数据字段让用户填写
- 如果当前评测集中为空，平台会提供4个常用的数据字段给用户填写
- 注意：ID、创建者、创建时间--这些字段是默认生成，不支持修改

#### 3.2.2 批量上传

在评测集详情页面，选择新增数据项 > 批量上传来添加评测数据

- 注意：如果上传数据中的中文展示为乱码，则需要先把要上传的文件保存为CSV UTF-8格式、再上传
- 如果当前评测集中已有数据，则需要用户在上传之后，配置新上传列与已有列之间的映射关系

#### 3.2.3 AI扩充

在评测集详情页面，选择 AI扩充 基于已有数据生成新的数据（行）

- 用户需要选择模型并设置输出行数
- 点击"开始扩充"后，会开始执行；当前评测集页面会展示执行进度

### 3.3 列新增

#### 3.3.1 手动添加列

在评测集详情页面，选择 列管理 > 添加列

- 同时也可在这里修改已有列的名称
- 注意：ID、创建者、创建时间--这些字段是默认生成，不支持修改

#### 3.3.2 AI数据加工

在评测集详情页面，选择AI数据加工可基于已有数据生成新的数据（列）

- 用户需要选择模型并输入prompt，prompt通常是指定根据当前已有的哪一列生成新的列（即分类打标）
- AI数据加工能力可用于非常多场景：
  - 质量打标
  - 内容提取
  - 内容生成
  - 内容转换
  - 情感打标
