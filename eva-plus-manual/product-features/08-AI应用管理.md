# AI应用管理

---

评测过程中，可能需要实时调用AI应用，将必要信息传给AI应用，获取输出并进行评测。本文指导如何在Eva+平台创建并管理AI应用。

这里的AI应用管理支持两种类型的应用的创建及管理：

- **通用AI Agent**：不局限于某一垂直领域（如软件工程或编程）的智能体形态，基于大型语言模型（LLM）的理解与推理能力，能够接收自然语言指令，对任务进行拆解、规划与决策，并根据需要灵活调用不同工具或系统（如搜索、数据分析、应用操作、工作流编排等），以多步骤、跨系统的方式完成目标。相较于 code agent 聚焦于代码与工程流程，通用 Agent 更强调跨领域的泛化能力与任务协同能力，典型应用包括信息调研、数据整理、内容生成、业务流程自动化、个人助理与企业智能助手等。

- **Code Agent**：应用于软件工程和编程领域，利用大型语言模型（LLM）的推理能力，能够接收自然语言指令，并能动态地指导自身流程和工具的使用，以完成复杂的、多步骤的编程任务，包括代码的生成、修改、调试和测试等。如cursor、通义灵码等。

## 一、通用Agent

### 1.1 平台操作

1. 进入AI应用管理页面，点击新增AI应用，在抽屉中选择通用Agent创建

2. 有两种创建方式：

   **a. 没有创建过AI应用代码仓库**
   - 点击仓库输入框右上角的没有代码仓库？ 帮我创建 >
   - 输入项目名称及AI应用名称、描述
   - 点击去创建或确定，平台会根据设定好的内容初始化一个AI应用仓库，并在平台创建一个AI应用
   - 前往对应的仓库空间，按照规范根据自己的AI应用进行配置
   - 返回平台，确认创建的AI应用同步情况

   **b. 已根据1.2 git仓库规范创建好git仓库，注意新增versions/1.0.0分支，versions/xxx用来标记版本号**
   - 输入git链接，点击解析仓库
   - 如果一个仓库中含有多个应用，需要选择一个应用进行导入（一次仅支持导入一个应用）
   - 选择应用后，会自动同步当前git仓库下的参数配置，参数信息及初始化字段默认读取gitlab中的设置，可进行确认，在发起评测任务的过程中可以进行针对性修改
   - 点击确定完成AI应用创建，如果报错请务必增加 线上账号evaplus加入到git仓库管理员

### 1.2 git仓库规范

#### 1.2.1 分支命名规范

发布版本时，分支命名格式为 versions/版本号，例如：

- versions/1.0.0
- versions/1.1.0
- versions/2.0.0

注意：仓库中至少需要存在一个 versions/ 格式的分支，否则 Evaplus 平台无法创建该应用。

## 二、Code Agent

### 2.1 平台操作

1. 进入AI应用管理页面，点击新增AI应用

2. 在唤出的抽屉中可以选择
   - a. 引入Eva+管理的公共应用
   - b. 自定义应用，为保障自建应用可以顺利被平台读取并在沙箱运行，自定义应用需要按照Eva+的规范进行编写git仓库后传入

3. 应用创建完成后，如果存在配置信息，可以选择应用卡片页的详情进行编辑，初始化的配置信息可以保障评测任务的正常运行，在发起任务的时候也可以做临时调整
   - a. 如果后续还有新增版本，为避免多次配置，可以选择更新【当前版本引为模板】，后续版本如存在同样的配置字段会自动引入
   - b. 如果初始化应用就有1个以上的版本，各版本配置趋同的话，可以点击同步当前配置到所有版本避免初始化多次配置

4. 如果需要基于当前AI应用发起评测，除了可以在评测任务模块通过新增点选，也可以在应用管理直接选择【评测】

### 2.2 git仓库规范

**样例仓库**: https://code.alibaba-inc.com/evaplus-agents-public/evaplus-agent-demo/tree/main

#### 2.2.1 文件&目录说明

- **meta.yaml**：定义install.sh脚本和start.sh执行需要的参数规则，会被Eva+平台解析，用于编辑和下发Agent安装和运行参数。

- **install.sh**：执行agent的安装脚本，将agent安装到当前case虚拟机执行环境中，并设置好当前agent运行所需的参数，evaplus会将配置信息以env参数的形式进行传入，这个配置会跟meta.yaml定义的一致。

- **start.sh**：运行agent，执行实际的任务，比如写代码、回答问题等。返回的内容应该响应本身（如有），对agent运行信息（包括命令行输出、日志文件等）进行解析，返回本次运行的相关统计信息，比如token消耗，耗时等，输出到标准终端的格式应符合约定的json规范。

**约定输入**: PROMPT_FILE参数为约定参数，标识PROMPT内容的文件目录，start.sh可以通过读取该文件获取PROMPT内容

**约定输出**: 保障Agent能正常运行即可, 其它统计类输出格式待定

#### 2.2.2 agent版本号管理

这里使用git branch进行版本管理，所有以versions/{versionId}（比如versions/v1.0.2）形式命名的branch均被会在evaplus的产品上作为版本进行管理，当评测界面选择了对应的版本号后，评测脚本将切换为对应分支的agent脚本进行安装和运行。

**案例代码库**：https://code.alibaba-inc.com/evaplus-agents-public/qwen-code/tree/versions/1.0.0-beta

#### 2.2.3 需支持的系统

因为agent需要在评测集的每个case中进行现场安装和执行，而每个case底层的基础操作系统和版本可能不尽相同，因此agent的安装脚本可能需要适配预期会运行的case的不同底层操作系统。比如有的case底层是运行的是ubuntu 22上，那么这个agent的安装脚本就应同时支持在ubuntu 22上安装，否则可能就会失败。如果有的case底层运行的centos 8，那这个agent的安装脚本也应该同时支持在centos 8上安装。

一般来说，主流要适配的系统有：ubuntu:24.04。
