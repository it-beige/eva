# 评估指标

---

在评测过程中，评估指标充当裁判的角色，通过量化评测对象的输出结果来评估其表现。在发起评测任务前，至少要确认选择什么评估指标。本文指导在Eva+中创建和管理评估指标。

## 一、评估指标概述

评估指标是用于自动化或半自动化评估 AI Agent 效果的工具。评估指标通过预定义的规则，对评估对象的输出进行多维度分析，生成可量化的指标和归因结论。

目前，Eva+支持：

- **LLM 类型评估指标**：通过大模型判断预期输出和实际输出的差异，用于模型自动化评测的场景。评测的标准和评分由大模型的 Prompt 来定义。
- **Code 类型评估指标**：通过执行代码函数来对比预期输出和实际输出，检查输出是否包含特定的文本或是否符合特定的格式要求。适用于对评测的精确性要求比较高的场景，例如 JSON 格式判断等。

Eva+内置了多种评估指标Prompt 模板，您可以直接使用这些模板用于评估任务。

在执行评测任务时，LLM 评估指标会根据 Prompt 中预设的标准和规则对评估对象的输出进行打分，并提供得分原因。

## 二、创建评估指标

### 2.1 UI创建

#### 2.1.1 LLM类型指标

选择评估指标，点击新建评估指标，选择【LLM类型指标】，输入指标名称、描述以及Prompt。

如果没有灵感，可以点击【选择模板】，选择平台内置的公共指标进行更改，优化为更适合自己AI应用评测的指标。

**Prompt建议：**

- 输入评估裁判的定义内容。建议包括
- **角色**：定义评估裁判的角色及要评测的主要指标
- **评估维度**：说明评估指标应该从哪些方面来评估
- **评分标准**：说明应该采用什么样的评分制，3分制/5分制/扣分/加分……，不同的分值代表什么
- **评分清单**：说明评分具体的分数应该怎么结合评估维度打
- **变量**：按{{name}}的形式输入变量

#### 2.1.2 Code类型指标

选择【code类型指标】，如果要在平台上管理code类型指标，就需要在gitlab库按照平台要求的模板创建，模板如下：

**模版工程**：https://code.alibaba-inc.com/evaplus-metrics/evaplus-metrics-code-template

**代码规范描述**(具体参考上面的模板工程)

1. 代码结构要求, 必须是src/metrics/{业务自定义metric的名字}/metric.py
2. metric.py的实现必须是CustomMetric类, 并且必须实现"score"方法
3. 完成对应的meta.yaml

️如果没有在gitlab创建指标库，可以直接选择【未在gitlab创建】，由平台进行初始化后进一步编辑修改，也可以直接选择【已在gitlab创建】并导入解析指标。

## 三、公共评估指标说明

Eva+基于合作的专业评测团队，提供相当规模的内置评估指标以供使用。

### 3.1 LLM类型指标

即使用llm模型作为第三方裁判，基于输入信息对AI应用的结果进行指标分析。

1. 单轮指标
2. 多轮指标
3. 上下文连贯性
4. 用户挫败感
5. 任务完成度

### 3.2 Code类型指标

即通过执行代码函数来对比预期输出和实际输出，检查输出是否包含特定的文本或是否符合特定的格式要求。适用于对评测的精确性要求比较高的场景，例如 JSON 格式判断等。

- **等于（Equals）**：检查输出是否与期望的字符串完全匹配，即检测output与reference是否匹配。
- **正则匹配（Regex match）**：检查输出是否匹配指定的正则表达式模式。
- **包含（Contains）**：检查输出是否包含一个特定的子字符串。
- **JSON格式**：检查输出是否是一个有效的 JSON 格式。
- **Levenshtein**：检查输出与参考之间编辑距离的有效性。衡量将一个字符串转换成另一个字符串所需的最小单字符编辑次数（插入、删除或替换）的相似度百分比。
