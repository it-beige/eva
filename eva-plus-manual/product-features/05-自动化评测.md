# 自动化评测

---

应用上线后，在上报的大量 Trace 数据中，人工进行查看、筛选、回流将变得繁琐与不现实。

Eva+支持用户基于 Trace 数据设置自动化任务，允许在特定时间范围内，自动采样 Trace 数据，获取输入、输出并进行在线评测，旨在帮助开发者在应用发布到线上后的运维过程中，及时了解应用质量、洞察问题并进行优化，降低人工干预成本。

## 一、应用场景

- **线上质量监控**：应用部署上线后，用户在 Trace 模块设定自动化评测规则，Eva+将对应用的真实输出进行评测实验，并在 Trace 列表和详情页展示评测结果。通过自动评测结果，能够发现 AI 应用在面对部分 Query（Input）时，表现（Output）不佳，从而进行线上应用调优。

- **迭代效果比对**：在 AI 应用迭代的过程中，需要常态化了解迭代后的应用版本表现是否更加优异，用户在平台设定自动评测任务后，能够在持续查看该任务下，不同时间周期内、同一评估指标的的评测结果，进而比对迭代效果。

- **提升数据质量**：通过筛选自动评测中的低分 Trace，并回流成评测集，能够将线上真实数据沉淀为评测基准，不断丰富评测数据库，覆盖更多的场景和边界情况，提升评测的全面性和准确性。

---

## 二、创建自动化评测任务

创建自动化评测任务的页面分为两块，左侧为配置任务内容区，右侧为调试区。创建好一份自动化评测任务后，如果想确认规则是否可以获得想要的结果，可以通过右侧调试区确认。

### 2.1 创建任务

#### 1. 基本信息

- a. **名称**：auto评测任务的名称，名称不允许与已有自动化任务名称重复。后续产出的每天或每周的自动化任务名称会按照自动化任务名称+日期的方式组合；
- b. **状态**：用来配置自动化任务是否开启，默认为开启；

#### 2. 过滤采样规则

用来配置基于什么样的规则对线上的Trace数据进行过滤筛选及采样。

- a. **过滤条件**：通过过滤器筛选符合要求的 Trace 数据，只有符合要求的 Trace 数据才会被自动评测任务采集，目前支持的配置方式见下表：

| 字段      | 操作符                         | 值                 |
| --------- | ------------------------------ | ------------------ |
| TraceID   | =                              | 填写框：字符串     |
| 耗时      | 不为空/为空, 大于等于/小于等于 | 填写框：数字       |
| 状态      | 属于                           | 成功/失败          |
| 输入Token | 大于等于/小于等于/=            | 填写框：数字       |
| 输出Token | 大于等于/小于等于/=            | 填写框：数字       |
| 总Token   | 大于等于/小于等于/=            | 填写框：数字       |
| Input     | 为空/不为空, 包含/不包含       | 填写：任意         |
| Output    | 为空/不为空, 包含/不包含       | 填写：任意         |
| Metadata  | >=<, 包含/不包含, 为空/不为空  | 填写框：数字或任意 |

- b. **采样率**：采样的比例，100% 表示全采样，即符合筛选范围的 Trace 数据都会被采样。
- c. **运行时间**：选择时间范围，只有该时间范围内上报的 Trace 才会被自动评测任务采集。只能选择当前时间戳之后的时间点，不支持选择过去的时间点。
- d. **重复频率**：自动评测任务的重复频率，支持设置为天或周的维度重复运行。
  - 天：每天产出前一天评测结果
  - 周：每周一产出上一周评测结果
- e. **单次上限**：单次采样上限，设置上限以避免大量采样导致超支。总上限（自动化评测中的所有评测任务的数量）为1000

#### 3. 评测规则

配置方式与发起评测任务类似，配置评估指标→评估模型→字段映射即可，需要注意的是字段映射目前默认映射Trace的input、output。

- 支持LLM类型指标、Code类型指标

### 2.2 调试任务

任务配置完成后，可以使用调试功能确认是否可以过滤采样到对应的数据以及评测规则是否可以生效。

#### 1. 调试过滤采样规则

选择当前时间戳之前的时间点，最多支持选择3天，选择完成后，即可点击调试运行，若过滤规则合理，则可以看到展示的示例数据。

#### 2. 调试评测规则

默认取过滤采样调试中的第一条数据作为评测规则调试对象，点击调试运行，即可看到评测的结果示例展示。

---

## 三、管理及查看评测结果

配置完成后，评测将按照配置的重复频率在固定日期产出评测任务，点击即可跳转到对应的评测任务详情查看评测结果。

- **周期性调度生成评测报告**
- **点击"详情"查看评测任务分析**
