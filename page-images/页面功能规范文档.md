## 1. 模块概述

### 1.1 模块定位
项目列表模块是 Eva+ 平台的核心管理模块，用于统一管理 AI 应用项目的创建、编辑、查看和删除操作。支持多种项目创建方式，提供完善的项目信息管理功能。

### 1.2 核心功能
- ✅ 项目列表展示与检索
- ✅ 三种项目创建方式（关联应用/直接创建/联合创建）
- ✅ 项目信息编辑与管理
- ✅ 项目权限管理（管理员/普通用户）
- ✅ 项目删除操作

### 1.3 用户角色
- **平台管理员**: 可查看所有项目、创建/编辑/删除任意项目
- **项目管理员**: 可管理自己负责的项目
- **普通用户**: 仅可查看被授权的项目

| 页面编号 | 页面名称 | 路由路径 | 功能说明 |
|---------|---------|---------|---------|
| P01 | 项目列表页 | `/projects` | 展示所有项目，支持搜索和筛选 |

**B. 页面标题区**
- **页面标题**: "项目列表"（一级标题，左侧对齐）
- **操作按钮**: "创建项目"（蓝色主按钮，右侧对齐）

**C. 搜索过滤区**
- **搜索框**: 
  - Placeholder: "支持项目ID、pid、项目名称搜索"
  - 右侧搜索图标 🔍

**D. 数据表格区**

**表格列定义**（从左到右）:

| 列名 | 字段名 | 数据类型 | 显示规则 | 排序 | 筛选 | 说明 |
|-----|-------|---------|---------|-----|-----|------|

| 项目名称 | `projectName` | String | 普通文本 | ✅ | ❌ | 最多显示20字符，超出省略 |
| pid/统一身份 | `pid` | String | 普通文本 | ❌ |  | 项目的唯一标识符 |
| 应用code | `appCode` | String | 普通文本 | ❌ | ❌ | 关联的应用编码 |
| 项目来源 | `source` | Enum | 标签显示 | ❌ | ✅ | IDEALAB/IDEALAB_WORKSPACE/DEMO等 |
| 项目描述 | `description` | String | 普通文本 | ❌ | ❌ | 最多显示30字符，超出省略 |
| 管理员 | `admins` | Array | 文本列表 | ❌ | ✅ | 逗号分隔，超过3人显示"等X人" |
| 用户数 | `userCount` | Number | 数字 | ✅ | ❌ | 项目关联的用户总数 |
| 创建时间 | `createTime` | DateTime | YYYY-MM-DD HH:mm:ss | ✅ | ❌ | 列表-2中显示的额外列 |
| 操作 | - | - | 按钮组 | ❌ | ❌ | 编辑/删除按钮 |

**表格交互规则**:
- **行高亮**: 鼠标悬停时行背景色变化
- **分页**: 默认每页20条，支持 10/20/50/100 条切换
- **排序**: 点击列头可排序（升序/降序/取消）
- **空状态**: 无数据时显示空状态插图和提示文字
- **加载状态**: 数据加载时显示骨架屏或 Loading 动画

**E. 操作列按钮**

| 按钮 | 权限要求 | 交互行为 | 样式 |
|-----|---------|---------|------|

| 删除 | 项目管理员或平台管理员 | 弹出删除确认对话框 | 红色文字链接 |

**删除确认对话框**:

#### 3.1.4 特殊交互逻辑

1. **搜索联动**: 
   - 输入关键词后实时过滤（防抖 300ms）
   - 支持多条件组合搜索（项目ID + 项目名称）

2. **管理员显示优化**:
   - 管理员数量 ≤ 3: 全部显示，逗号分隔
   - 管理员数量 > 3: 显示前3个 + "等X人"
   - 鼠标悬停显示完整管理员列表（Tooltip）

3. **项目来源标签**:
   - IDEALAB: 蓝色标签
   - IDEALAB_WORKSPACE: 绿色标签
   - DEMO: 灰色标签
   - 其他: 默认蓝色标签

4. **权限控制**:
   - 非管理员用户: 操作列隐藏或禁用
   - 平台管理员: 可操作所有项目
   - 项目管理员: 仅可操作自己管理的项目

#### 3.2.3 表单字段清单

| 字段名 | 字段标签 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|-------|---------|------|------|-------|---------|------|
| `createMode` | 新建方式 | Radio Group | ✅ | `direct` | 三选一 | 关联应用/场景、直接创建、联合创建 |
| `pid` | pid | Input | ✅ | 自动生成 | 必填，唯一性校验 | 项目唯一标识，系统自动生成 |

#### 3.2.4 交互逻辑

1. **新建方式切换**:
   - 切换时清空不相关字段
   - 关联应用模式: 显示"平台选择"和"关联应用"字段
   - 联合创建模式: 显示"联合范围"字段
   - 直接创建模式: 显示"pid"字段

2. **pid 自动生成**:
   - 进入页面时自动生成唯一 pid
   - 格式: 小写字母+数字组合（如 `ug6cpy6y`）
   - 不可编辑（只读状态）

3. **管理员选择**:
   - 默认选中当前登录用户
   - 支持搜索添加其他用户
   - 显示格式: `姓名(工号)`
   - 支持删除已选用户（× 按钮）

4. **普通用户选择**:
   - 搜索框 placeholder: "搜索并添加用户"
   - 支持模糊搜索（姓名/工号）
   - 多选模式

5. **表单验证**:
   - 必填项未填: 红色边框 + 错误提示
   - 项目名称重复: 提示"该项目名称已存在"
   - 实时验证: 失焦时触发校验

6. **提交逻辑**:
   - 点击"创建项目"按钮
   - 前端校验通过后提交
   - 显示 Loading 状态
   - 成功后跳转至项目列表页
   - 失败后显示错误提示

#### 3.2.5 右侧步骤提示区

**显示内容**:
```
01 | 填写项目基础信息
```

**样式**:
- 浅灰色背景
- 蓝色高亮当前步骤
- 后续步骤灰色（不可点击）

   - 项目管理员: 可编辑所有可编辑字段
   - 平台管理员: 同项目管理员

5. **保存逻辑**:
   - 点击"保存项目"按钮
   - 前端校验
   - 提交 PUT 请求更新项目信息
   - 成功后跳转至项目列表页
   - 失败后显示错误提示，停留在当前页

```typescript
// 权限判断工具函数
function canEditProject(project: Project, currentUser: User): boolean {
  // 平台管理员可编辑所有项目
  if (currentUser.role === 'PLATFORM_ADMIN') return true;
  
  // 项目管理员可编辑自己管理的项目
  return project.admins.some(admin => admin.userId === currentUser.userId);
}

function canDeleteProject(project: Project, currentUser: User): boolean {
  // 删除权限同编辑权限
  return canEditProject(project, currentUser);
}
```
